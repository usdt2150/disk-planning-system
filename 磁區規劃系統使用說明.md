# 磁區規劃系統使用說明

## 🚀 快速啟動流程

### 1. 啟動系統服務
```bash
# 方法一：使用批次檔（推薦）
雙擊執行：D:\磁區規劃\啟動系統.bat

# 方法二：手動啟動
# 開啟兩個終端機視窗：

# 終端機1 - 啟動API服務
cd D:\磁區規劃
python api.py

# 終端機2 - 啟動前端
cd D:\磁區規劃\frontend
npm run dev
```

### 2. 訪問系統頁面
- **前端系統**: http://localhost:3002
- **API服務**: http://localhost:5000
- **健康檢查**: http://localhost:5000/api/health
- **系統指標**: http://localhost:5000/api/metrics
- **DataEase BI**: http://localhost:8081 (需要等待啟動)

## 📋 系統功能說明

### API端點列表
- `GET /api/status` - 獲取系統狀態
- `GET /api/health` - 健康檢查
- `GET /api/metrics` - 系統性能指標
- `GET /api/files` - 獲取所有文件資訊
- `GET /api/files/<key>` - 獲取特定文件資訊
- `GET /api/disk-usage` - 獲取磁碟使用情況
- `POST /api/script/test` - 測試腳本
- `POST /api/script/execute` - 執行腳本
- `POST /api/backup/create` - 創建備份

### DataEase 整合功能
- `POST /api/dataease/collect-data` - 收集數據
- `POST /api/dataease/export-data` - 導出數據
- `GET /api/dataease/dashboard-config` - 獲取儀表板配置
- `GET /api/dataease/datasets/<name>` - 獲取數據集
- `GET /api/dataease/integration-guide` - 獲取整合指南

## 🛠️ 故障排除

### 常見問題
1. **端口被占用**
   - 檢查端口：`netstat -ano | findstr :5000`
   - 終止進程：`taskkill /PID <PID> /F`

2. **DataEase 無法啟動**
   - 檢查Docker狀態：`docker ps`
   - 重啟DataEase：`docker restart dataease`
   - 查看日誌：`docker logs dataease`

3. **前端無法訪問**
   - 檢查npm進程是否運行
   - 重新安裝依賴：`npm install`
   - 重新啟動：`npm run dev`

### 日誌位置
- API日誌：`D:\磁區規劃\api.log`
